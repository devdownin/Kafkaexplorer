Learn how to use Flink SQL to explore and analyze your Kafka streams

info Comment ça marche ?

Contrairement au SQL classique, les requêtes sur Kafka sont continues. L'application échantillonne les résultats et arrête automatiquement le job après 10 secondes ou 50 lignes. Vous pouvez arrêter manuellement une requête en cours avec le bouton Stop.

TYPE DE REQUÊTE OBJECTIF EXEMPLE
DDL source Kafka Déclarer un flux Kafka JSON avec event time + watermark 
CREATE TABLE user_events (
  user_id    STRING,
  event_type STRING,
  payload    STRING,
  event_time TIMESTAMP_LTZ(3),
  WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND
)
WITH (
  'connector' = 'kafka',
  'topic' = 'user-events',
  'properties.bootstrap.servers' = 'kafka:9092',
  'properties.group.id' = 'flink-sql-demo',
  'scan.startup.mode' = 'earliest-offset',
  'format' = 'json'
);
SELECT / WHERE Lecture filtrée du flux 
SELECT
  user_id,
  event_time,
  payload
FROM user_events
WHERE event_type = 'CLICK';
Agrégat simple Compter par type d’événement 
SELECT
  event_type,
  COUNT(*) AS nb_events
FROM user_events
GROUP BY event_type;
Fenêtre TUMBLE Agrégat par fenêtres fixes de 5 min 
SELECT
  window_start,
  window_end,
  event_type,
  COUNT(*) AS nb_events
FROM TABLE(
  TUMBLE(
    TABLE user_events,
    DESCRIPTOR(event_time),
    INTERVAL '5' MINUTE
  )
)
GROUP BY window_start, window_end, event_type;
